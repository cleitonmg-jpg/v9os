1. Visão Geral do Sistema
O que é o V9 OS?
É um sistema web completo para gerenciar uma oficina mecânica. Funciona pelo navegador (como um site), mas roda no computador da própria empresa sem precisar de internet. Permite cadastrar clientes, veículos, abrir ordens de serviço, gerar orçamentos, emitir PDFs e enviar mensagens pelo WhatsApp.

Como o sistema é dividido?

O sistema tem duas partes que trabalham juntas:

🖥️
Frontend (A Tela)
É a parte visual que você vê no navegador — botões, tabelas, formulários. Feito com React e TypeScript. Roda na porta 5173.
⚙️
Backend (O Servidor)
É a parte invisível que processa os dados, salva no banco e responde às ações. Feito com Node.js e Express. Roda na porta 3001.
🗄️
Banco de Dados
Um arquivo SQLite (dev.db) que fica na pasta do projeto. Guarda todos os dados: clientes, veículos, OS, etc. Gerenciado pelo Prisma ORM.
🔐
Autenticação JWT
Sistema de login com token seguro. Sem login, nenhuma tela é acessível. O token expira em 8 horas e é renovado a cada acesso.

2. Funcionalidades do Sistema

4. Banco de Dados
O que é o banco de dados?
É um arquivo chamado dev.db que fica na pasta raiz do projeto. Funciona como uma planilha Excel, mas muito mais rápido e seguro. Todos os dados do sistema ficam nele. O Prisma ORM é a ferramenta que faz o sistema conversar com esse arquivo.

Tabelas do Banco (arquivo: prisma/schema.prisma)

TabelaO que guardaCampos principais
Technician Técnicos e usuários do sistema username, name, senha (hash), cargo, ativo
Client Clientes da oficina nome, CPF/CNPJ (opcional), telefone, e-mail, endereço
Vehicle Veículos dos clientes placa, marca, modelo, ano, cor, chassi, km, clienteId
OSBudget Ordens de serviço e orçamentos número, tipo (OS/Orçamento), status, data, clienteId, veículoId, defeito, total, observações
OSItem Itens dentro de uma OS/orçamento descrição, quantidade, preço unitário, total, tipo (Serviço/Peça), técnicoId
ServiceItem Catálogo de serviços e peças descrição, tipo, preço de custo, preço de venda, estoque, ativo
AuditLog Registro de ações realizadas técnicoId, ação, entidade, data
⚠️ Atenção — Faça backup do arquivo dev.db!
Todos os dados ficam nesse único arquivo. Se ele for excluído ou corrompido, todos os dados são perdidos. Recomenda-se fazer cópia desse arquivo regularmente para um pendrive ou Google Drive.

5. Arquivos do Servidor (Backend)
O backend é o "motor" invisível do sistema. Ele recebe os pedidos da tela, processa, salva no banco e retorna o resultado. O usuário nunca vê essa parte diretamente.
⚙️
server/index.ts
BACKEND PRINCIPAL
Este é o coração do sistema. É um servidor que fica rodando continuamente na porta 3001 do computador. Ele recebe todas as requisições da tela (criar cliente, salvar OS, etc.), processa e responde com os dados. Está organizado em seções:
  • Conexão com banco: Usa Prisma + libsql para conectar ao arquivo dev.db
  • Middleware de autenticação: Toda rota protegida verifica o token JWT
  • POST /api/auth/login: Valida usuário e senha, gera token de 8 horas
  • CRUD de Clientes: Criar, listar, editar, excluir clientes
  • CRUD de Veículos: Criar, listar, editar, excluir veículos
  • CRUD de OS/Orçamentos: Inclui lógica de itens e cálculo de total
  • GET /api/stats: Retorna os números do Dashboard
  • CRUD de Técnicos: Inclui hash de senha ao criar/atualizar
  • CRUD de Catálogo: Serviços e peças com custo/venda/estoque
TecnologiaNode.js + Express 5 + TypeScript
Porta3001 (http://localhost:3001)
SegurançaJWT (token expira em 8h)
BancoPrisma ORM + SQLite via libsql
📐
prisma/schema.prisma
BANCO DE DADOS
Define a estrutura (esquema) do banco de dados. É como um projeto arquitetônico — descreve quais tabelas existem, quais colunas cada uma tem e como elas se relacionam. Quando este arquivo é alterado, é necessário rodar uma "migração" para aplicar as mudanças no banco real.
Banco utilizadoSQLite (arquivo local)
Comando de migraçãonpx prisma migrate dev

7. Componentes Visuais (src/components/)
O que são componentes?
São "peças" de tela reutilizáveis. Em vez de escrever o mesmo código toda vez, cria-se um componente uma vez e usa-se em vários lugares. Por exemplo, o Modal (janela flutuante) é usado em todas as telas do sistema.
🏗️
src/components/Layout.tsx
COMPONENTE
Define a estrutura geral de todas as telas após o login. Organiza a tela em: menu lateral à esquerda (Sidebar) + área de conteúdo principal à direita. Exibe no topo o nome e as iniciais do usuário logado. Toda página do sistema (Dashboard, Clientes, etc.) é exibida dentro deste layout.
📌
src/components/Sidebar.tsx
COMPONENTE
O menu de navegação lateral fixo à esquerda. Lista os itens do menu na ordem configurada e destaca visualmente o item da página atual. Contém o botão "Sair" no rodapé que encerra a sessão. Para alterar a ordem ou nome dos itens do menu, este é o arquivo a editar.

9. Rotas da API (Comunicação Tela ↔ Servidor)
O que são rotas de API?
São os "endereços" que a tela usa para pedir informações ao servidor. Toda operação (criar cliente, buscar OS, etc.) passa por uma dessas rotas. Todas (exceto login) exigem o token JWT no cabeçalho da requisição.

Autenticação

MétodoRotaO que faz
POST/api/auth/loginValida usuário e senha, retorna token JWT. Não exige autenticação.

Clientes

MétodoRotaO que faz
GET/api/clientsRetorna lista de todos os clientes com seus veículos
POST/api/clientsCria novo cliente. CPF/CNPJ vazio é salvo como nulo.
PUT/api/clients/:idAtualiza dados de um cliente pelo ID
DELETE/api/clients/:idRemove um cliente do banco

Veículos

MétodoRotaO que faz
GET/api/vehiclesLista todos os veículos com dados do cliente vinculado
POST/api/vehiclesCadastra novo veículo. Placa deve ser única.
PUT/api/vehicles/:idAtualiza dados do veículo
DELETE/api/vehicles/:idRemove veículo

Ordens de Serviço / Orçamentos

MétodoRotaO que faz
GET/api/osLista todas as OS e Orçamentos com cliente, veículo e itens
POST/api/osCria nova OS/Orçamento. Gera número sequencial automático.
GET/api/os/:idBusca uma OS específica pelo ID
PUT/api/os/:idAtualiza OS. Apaga e recria os itens inteiros.
DELETE/api/os/:idRemove OS e seus itens (cascade)

Técnicos, Estatísticas e Catálogo

MétodoRotaO que faz
GET/api/statsRetorna totais para o Dashboard (orçamentos, OS, veículos)
GET/api/techniciansLista técnicos (sem expor senha)
POST/api/techniciansCria técnico com senha criptografada (bcrypt)
PUT/api/technicians/:idAtualiza técnico. Se nova senha informada, re-criptografa.
DELETE/api/technicians/:idDesativa técnico (não exclui, preserva histórico)
GET/api/catalogLista itens do catálogo ordenados por descrição
POST/api/catalogCria item no catálogo
PUT/api/catalog/:idAtualiza item do catálogo
DELETE/api/catalog/:idExclui item do catálogo

10. Como Iniciar o Sistema
⚠️ Pré-requisito
É necessário ter o Node.js instalado no computador. Para verificar, abra o Prompt de Comando e digite: node --version. Se aparecer um número (ex: v20.0.0), está instalado.

Primeira execução (instalação)

# 1. Instalar dependências
npm install

# 2. Criar o banco de dados
npx prisma migrate dev

# 3. Criar usuário admin (user/user123) e dados de exemplo
node prisma/seed.cjs

Uso diário

Abrir dois terminais (Prompt de Comando) na pasta do projeto:

# Terminal 1 — Servidor backend (porta 3001)
npm run server

# Terminal 2 — Frontend (porta 5173)
npm run dev
✅ Depois de subir os dois servidores
Abrir o navegador e acessar: http://localhost:5173
Fazer login com: usuário user / senha user123

12. Glossário — Termos Técnicos Explicados
React
Biblioteca JavaScript para criar interfaces visuais. Divide a tela em "componentes" reutilizáveis (como peças de Lego).
TypeScript
Versão do JavaScript com tipagem — define que um campo "idade" só pode ser número, evitando erros. O código é "traduzido" para JavaScript antes de rodar.
Node.js
Permite rodar JavaScript no computador (não só no navegador). É a base do servidor backend.
Express
Framework para Node.js que facilita criar servidores HTTP. Define as rotas (endereços) que o frontend pode chamar.
Prisma ORM
Ferramenta que traduz comandos TypeScript em SQL, permitindo manipular o banco sem escrever SQL diretamente. ORM = Object-Relational Mapping.
SQLite
Banco de dados que fica em um único arquivo no HD (dev.db). Ideal para sistemas locais por ser simples e não precisar de servidor separado.
JWT
JSON Web Token — uma "chave digital" gerada pelo servidor após o login. A tela envia essa chave em cada pedido para provar que o usuário está autenticado.
API REST
Conjunto de regras para comunicação entre sistemas via HTTP. O frontend "chama" endereços (rotas) do backend para buscar ou enviar dados em formato JSON.
Axios
Biblioteca que facilita fazer chamadas HTTP no JavaScript. Usada no arquivo api.ts para comunicar o frontend com o backend.
Tailwind CSS
Framework de estilos CSS. Em vez de escrever CSS separado, as classes são aplicadas diretamente no HTML (ex: "bg-blue-500" deixa o fundo azul).
Vite
Ferramenta que serve o frontend em desenvolvimento (npm run dev) e empacota tudo para produção (npm run build). É rápido e moderno.
Migration (Migração)
Arquivo SQL gerado quando o schema do banco é alterado. Registra o histórico de mudanças e atualiza o banco de dados para a nova estrutura.
Hash de senha
As senhas nunca são salvas em texto puro no banco. São transformadas em um código irreversível (hash) usando bcrypt. Mesmo quem acessa o banco não consegue descobrir a senha original.
localhost
Endereço que aponta para o próprio computador. "localhost:5173" significa: porta 5173 neste mesmo computador.
Context (React)
Mecanismo para compartilhar dados entre componentes sem passar manualmente por cada um. O AuthContext compartilha os dados de login com todo o sistema.
CRUD
Create, Read, Update, Delete — as quatro operações básicas de qualquer sistema: criar, ler, editar e excluir registros.
z-index
Propriedade CSS que define qual elemento fica "em cima" de outro na tela. Número maior = fica na frente. O picker de catálogo usa z-60 para ficar acima do modal (z-50).
localStorage
Área de armazenamento do navegador (como um "post-it" digital). O token JWT é salvo aqui para que o login persista após fechar e reabrir o navegador.
